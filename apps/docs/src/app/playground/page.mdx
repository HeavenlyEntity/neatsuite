# Interactive Playground

Experiment with the NeatSuite HTTP client concepts in a safe sandbox.  
Edit the code on the left and press **RUN** to see the result on the right.  
This page performs **no network calls** and uses **no credentials**.

<Note type="warning">
  <strong>Learning demo only.</strong> Runs entirely in your browser (no requests). Do <strong>not</strong> paste real credentials.
  Real OAuth signing must be done <strong>server-side</strong> to keep secrets secure.
</Note>

## How to use this playground

1. **Edit the code** on the left (the example shows the basic structure).
2. Define a `main()` function that returns a value.
3. Press **RUN** to see the formatted JSON result.

## Understanding NetSuite OAuth 1.0a

When using NeatSuite’s **server-side** client, you typically configure:
- `consumerKey`, `consumerSecret`
- `tokenKey`, `tokenSecret`
- `realm` (your NetSuite account ID)
- `accountId` (if required by the client)

Store these in `.env.local` and use them **only on the server** (never in the browser).
